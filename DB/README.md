# Проектирование и реализация БД "GameStore"

Проект представляет собой реалзацию базы данных для магазина игр

## Технологический стек
* **СУБД:** Microsoft SQL Server 2025
* **Среда развертывания:** Docker
* **Инструменты:** SQL Server Management Studio (SSMS)

---

## Архитекнурная базы данных (Предметная область)
Выбранная областью является "Игровая индустрия". База данных моделируют работу магазина, учитывая связи между студиями, играми и пользователем.

### Состав таблиц и типы данных:
База включает 5 таблиц, использующих следующие типы данных:
 `UNIQUEIDENTIFIER` (GUID), `NVARCHAR` (строки), `DECIMAL` (числа с запятой),`DATETIME2` (дата/время), `BIT` (булево).

1.  **Studios** — Студии-разработчики (Связь **1:N** с таблицей Games).
2.  **Games** — Каталог игр (Цена, дата выхода, мультиплеер).
3.  **Genres** — Справочник жанров.
4.  **GameGenres** — Связующая таблица для реализации связи **N:N** (одна игра — много жанров).
5.  **Players** — Таблица пользователей (Никнейм, баланс, статус бана).

## Оптимизация и Индексы
В таблице `Players` создан **Уникальный индекс** на поле `Nickname`.

* **Обоснование:** Никнейм является ключевым идентификатором игрока. Уникальный индекс предотвращает дублирование имен на уровне БД и ускоряет поиск профиля при авторизации.
* **Минусы:** Незначительное замедление операций `INSERT`, так как СУБД должна проверять уникальность каждого нового значения.

Состав репозитория (Инструкция по развертыванию)

В репозитории подготовлены три ключевых SQL-скрипта:

1.  **[schemas.sql](./schemas.sql)** — DDL скрипт. Создает структуру базы "с нуля" (таблицы, ключи, связи).
2.  **[script.sql](./script.sql)** — Полный SQL-дамп (экспортированная база). Содержит и структуру, и тестовые данные для мгновенного развертывания.
3.  **[queries.sql](./queries.sql)** — Набор практических запросов:
    * Выборка с фильтрацией и сложной сортировкой.
    * Группировка данных (агрегатные функции).
    * Сложные JOIN-запросы (объединение 4-х таблиц).
    * Примеры команд UPDATE и DELETE.
