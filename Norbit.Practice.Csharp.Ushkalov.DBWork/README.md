## GameStore Management Utility
Консольная утилита для управления базой данных игрового магазина. Проект демонстрирует гибридный подход к работе с данными, используя Entity Framework Core для стандартных операций и ADO.NET для прямой работы с SQL-запросами и аналитики.

## 🛠 Технологический стек
* **Язык:** C# (.NET 8.0)
* **ORM:** Entity Framework Core (SQL Server Provider)
* **Data Access:** ADO.NET (Microsoft.Data.SqlClient)
* **СУБД:** Microsoft SQL Server (в Docker-контейнере)
* **Конфигурация:** Microsoft.Extensions.Configuration (JSON)

## 🚀 Запуск БД
1. Запуск базы данных (Docker)
Для работы приложения необходимо поднять контейнер с SQL Server. Выполни команду в терминале:

```powershell
docker run -e "ACCEPT_EULA=Y" -e "MSSQL_SA_PASSWORD=YourStrongPassword123!" -p 1433:1433 --name gamestore-db -d mcr.microsoft.com/mssql/server
(Замените YourStrongPassword123! на ваш реальный пароль).
```
## Подготовка структуры БД
2. После запуска контейнера выполните SQL-скрипты через SSMS или Azure Data Studio:

* schema.sql — создание таблиц и связей.
* script.sql — наполнение базы тестовыми данными.

## Настройка приложения
Отредактируйте файл appsettings.json в корне проекта:

* Укажите адрес сервера (по умолчанию 127.0.0.1,1433).

* Вставьте ваш пароль в строку подключения.

## 🏗 Архитектура проекта
Проект реализован с использованием паттерна Repository, что позволяет разделить бизнес-логику и логику доступа к данным:

* Models/ — POCO-классы, описывающие сущности БД (Game, Studio, Player, Genre).

* Data/GameStoreContext — контекст базы данных EF Core с настроенным Fluent API для связей "многие-ко-многим".

* Data/EfRepository — реализация CRUD-операций через ORM.

* Data/AdoRepository — реализация сложных отчетов и массовых обновлений через «сырой» SQL.

## 📈 Особенности реализации
* Параметризация: Адрес сервера СУБД вынесен в отдельную секцию конфигурации и подставляется в строку подключения динамически.

*Безопасность: Реализована параметризация SQL-запросов в ADO.NET для защиты от SQL-инъекций.

* Целостность данных: В EF Core настроены проверки существования внешних ключей (например, проверка StudioID перед добавлением игры).
